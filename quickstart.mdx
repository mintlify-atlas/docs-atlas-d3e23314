# Quick Start

This guide will walk you through creating your first working example with Epic Online Services Godot. By the end, you'll have a game that initializes the EOS SDK, authenticates a user, and fetches their display name.

## Get Your EOS Credentials

Before you begin, you'll need credentials from the Epic Developer Portal.

<Steps>
  <Step title="Log in to Epic Developer Portal">
    Visit the [Epic Developer Portal](https://dev.epicgames.com/portal) and log in with your Epic Games account.
  </Step>
  <Step title="Create or Select a Product">
    Either create a new product or select an existing one from your organization.
  </Step>
  <Step title="Copy Your Credentials">
    Navigate to **Product Settings** and copy the following credentials:
    - Product ID
    - Sandbox ID
    - Deployment ID
    - Client ID
    - Client Secret
  </Step>
  <Step title="Configure Client Policy (Important)">
    Go to **Product Settings → Client Policies**:
    - Select **Custom policy** for the Client policy type
    - Enable **User is required**
    - Enable all features and actions (for development)
    
    <Warning>
      In production, you should grant minimal permissions based on what your game actually needs.
    </Warning>
  </Step>
  <Step title="Enable Epic Account Services Permissions">
    Go to **Epic Account Services → Permissions** and enable:
    - Basic Profile
    - Online Presence
    - Friends
  </Step>
</Steps>

<Note>
  Keep your credentials secure! Never commit them to version control. Consider using environment variables or Godot's encryption features.
</Note>

## Create Your First EOS Script

Let's create a simple "Hello EOS" script that demonstrates the core functionality.

<Steps>
  <Step title="Create a New Scene">
    In Godot, create a new scene with a `Node` as the root. Save it as `Main.tscn`.
  </Step>
  <Step title="Attach a Script">
    Attach a new script to the root node and name it `Main.gd`.
  </Step>
  <Step title="Add the Hello EOS Code">
    Replace the script contents with the following code:

```gdscript Main.gd
extends Node

# Replace these with your actual credentials from Epic Developer Portal
const PRODUCT_NAME = "My Awesome Game"
const PRODUCT_VERSION = "1.0"
const PRODUCT_ID = "YOUR_PRODUCT_ID_HERE"
const SANDBOX_ID = "YOUR_SANDBOX_ID_HERE"
const DEPLOYMENT_ID = "YOUR_DEPLOYMENT_ID_HERE"
const CLIENT_ID = "YOUR_CLIENT_ID_HERE"
const CLIENT_SECRET = "YOUR_CLIENT_SECRET_HERE"

func _ready() -> void:
    print("Starting Hello EOS example...")
    
    # Setup logging
    HLog.log_level = HLog.LogLevel.INFO
    
    # Create credentials
    var credentials = HCredentials.new()
    credentials.product_name = PRODUCT_NAME
    credentials.product_version = PRODUCT_VERSION
    credentials.product_id = PRODUCT_ID
    credentials.sandbox_id = SANDBOX_ID
    credentials.deployment_id = DEPLOYMENT_ID
    credentials.client_id = CLIENT_ID
    credentials.client_secret = CLIENT_SECRET
    
    # Initialize EOS Platform
    print("Setting up EOS...")
    var setup_success := await HPlatform.setup_eos_async(credentials)
    if not setup_success:
        printerr("Failed to setup EOS. See logs for error details")
        return
    
    print("EOS setup successful!")
    
    # Get SDK version
    var sdk_constants := EOS.Version.VersionInterface.get_constants()
    print("EOS SDK Version: %s" % EOS.Version.VersionInterface.get_version())
    
    # Connect authentication signals
    HAuth.logged_in.connect(_on_logged_in)
    HAuth.login_error.connect(_on_login_error)
    HAuth.display_name_changed.connect(_on_display_name_changed)
    
    # Login anonymously (easiest method for testing)
    print("Logging in anonymously...")
    await HAuth.login_anonymous_async()

func _on_logged_in():
    print("✓ Login successful!")
    print("  Product User ID: %s" % HAuth.product_user_id)
    print("  Display Name: %s" % HAuth.display_name)
    print("\nHello EOS example completed successfully!")

func _on_login_error(result_code: EOS.Result):
    printerr("✗ Login failed: %s" % EOS.result_str(result_code))

func _on_display_name_changed():
    print("Display name updated: %s" % HAuth.display_name)

func _notification(what: int) -> void:
    if what == NOTIFICATION_WM_CLOSE_REQUEST:
        print("Shutting down EOS...")
        EOS.Platform.PlatformInterface.release()
        var res := EOS.Platform.PlatformInterface.shutdown()
        if not EOS.is_success(res):
            printerr("Failed to shutdown EOS: ", EOS.result_str(res))
```
  </Step>
  <Step title="Add Your Credentials">
    Replace the placeholder values (`YOUR_PRODUCT_ID_HERE`, etc.) with your actual credentials from the Epic Developer Portal.
  </Step>
</Steps>

## Run Your First EOS Example

<Steps>
  <Step title="Set as Main Scene">
    In Godot, go to **Project → Project Settings → Application → Run** and set `Main.tscn` as the Main Scene.
  </Step>
  <Step title="Run the Project">
    Press F5 or click the Play button to run your project.
  </Step>
  <Step title="Check the Output">
    Watch the Output panel at the bottom of the Godot editor. You should see:
    
    ```
    Starting Hello EOS example...
    Setting up EOS...
    EOS setup successful!
    EOS SDK Version: 1.18.1.2
    Logging in anonymously...
    ✓ Login successful!
      Product User ID: abc123...
      Display Name: 
    
    Hello EOS example completed successfully!
    ```
  </Step>
</Steps>

<Note>
  Anonymous login is the simplest authentication method and perfect for testing. The display name will be empty for anonymous users.
</Note>

## Understanding the Code

Let's break down what this example does:

### 1. Setup Credentials

```gdscript
var credentials = HCredentials.new()
credentials.product_name = PRODUCT_NAME
credentials.product_version = PRODUCT_VERSION
credentials.product_id = PRODUCT_ID
// ... more credentials
```

The `HCredentials` class holds all the credentials needed to connect to Epic Online Services. These identify your game and authorize it to use EOS features.

### 2. Initialize the Platform

```gdscript
var setup_success := await HPlatform.setup_eos_async(credentials)
```

The `HPlatform.setup_eos_async()` method initializes the EOS SDK and creates the platform instance. This must be done before using any other EOS features. The `await` keyword makes this asynchronous - the code waits for initialization to complete.

### 3. Connect to Signals

```gdscript
HAuth.logged_in.connect(_on_logged_in)
HAuth.login_error.connect(_on_login_error)
```

EOS uses Godot signals to notify your game about events. Here we're listening for successful login and login errors.

### 4. Authenticate the User

```gdscript
await HAuth.login_anonymous_async()
```

This initiates an anonymous login, which doesn't require any user credentials. It's perfect for testing and single-player games.

## Next Steps: Try Different Login Methods

Now that you have a working example, try different authentication methods:

### Login with DevAuth Tool (For Development)

The DevAuth tool is perfect for local development and testing:

<Steps>
  <Step title="Download DevAuth Tool">
    Download from [Epic's DevAuthTool](https://dev.epicgames.com/docs/services/en-US/DeveloperAuthenticationTool/index.html)
  </Step>
  <Step title="Run DevAuth Tool">
    Start the tool - it runs on `localhost:4545` by default
  </Step>
  <Step title="Add a Credential">
    Create a test credential (e.g., "Player1")
  </Step>
  <Step title="Update Your Code">
    Replace the login line with:
    
```gdscript
await HAuth.login_devtool_async("localhost:4545", "Player1")
```
  </Step>
</Steps>

### Login with Device ID

Device ID login is useful for mobile games and provides persistent user identity:

```gdscript
await HAuth.login_device_id_async("UniqueDeviceID", "PlayerDisplayName")
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Setup failed with error code">
    **Check your credentials** - Make sure all credentials from the Developer Portal are correct.
    
    **Check client policy** - Ensure you've configured the client policy to allow the features you're using.
    
    **Check the logs** - Set `HLog.log_level = HLog.LogLevel.DEBUG` for more detailed error messages.
  </Accordion>
  
  <Accordion title="Login fails immediately">
    **Anonymous login issues** - Make sure your client policy has the correct permissions.
    
    **DevAuth connection failed** - Ensure the DevAuth tool is running and accessible at `localhost:4545`.
    
    **Network issues** - Check your internet connection - EOS requires network access.
  </Accordion>
  
  <Accordion title="No output in console">
    **Check logging level** - Make sure `HLog.log_level` is set to `INFO` or `DEBUG`.
    
    **Check EOS logs** - You can also enable EOS SDK logging:
    
    ```gdscript
    HPlatform.log_msg.connect(func(msg): print("SDK: ", msg.message))
    HPlatform.set_eos_log_level(EOS.Logging.LogCategory.AllCategories, EOS.Logging.LogLevel.Info)
    ```
  </Accordion>
  
  <Accordion title="Product User ID is empty">
    This usually means authentication didn't complete successfully. Check for error messages in the console.
  </Accordion>
</AccordionGroup>

## Best Practices

<CardGroup cols={2}>
  <Card title="Secure Your Credentials" icon="lock">
    Never commit credentials to version control. Use environment variables or Godot's script encryption when exporting.
  </Card>
  
  <Card title="Handle Errors Gracefully" icon="triangle-exclamation">
    Always check for errors and provide user-friendly feedback when EOS operations fail.
  </Card>
  
  <Card title="Test Across Platforms" icon="mobile">
    EOS behavior can vary by platform. Test on all your target platforms early.
  </Card>
  
  <Card title="Use Proper Shutdown" icon="power-off">
    Always call `EOS.Platform.PlatformInterface.release()` and `shutdown()` when your game closes.
  </Card>
</CardGroup>
