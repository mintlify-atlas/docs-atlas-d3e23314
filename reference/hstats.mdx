## Overview

The `HStats` class provides methods for querying and updating player statistics. Stats are used for achievements, leaderboards, and tracking player progress.

## Methods

### get_stats_async

```gdscript
func get_stats_async() -> Array
```

Returns an Array of stats for the current user.

<ResponseField name="return" type="Array">
  Array of stat dictionaries. Each stat has:
  - `name`: String - name of the stat
  - `value`: int - current value for the stat
  - `start_time`: int - unix timestamp or `EOS.Stats.STATS_TIME_UNDEFINED`
  - `end_time`: int - unix timestamp or `EOS.Stats.STATS_TIME_UNDEFINED`
</ResponseField>

**Example:**

```gdscript
var stats = await HStats.get_stats_async()
for stat in stats:
    print("Stat: ", stat.name, " = ", stat.value)
```

### ingest_stat_async

```gdscript
func ingest_stat_async(stat_name: String, _ingest_amount: int) -> EOS.Result
```

Ingest a stat by the specified amount. This updates the stat value on the server.

<ParamField path="stat_name" type="String" required>
  The name of the stat to ingest
</ParamField>

<ParamField path="_ingest_amount" type="int" required>
  The amount to add to the stat value
</ParamField>

<ResponseField name="return" type="EOS.Result">
  Result code from the ingest operation
</ResponseField>

**Example:**

```gdscript
# Increment the "kills" stat by 1
var result = await HStats.ingest_stat_async("kills", 1)
if EOS.is_success(result):
    print("Stat updated successfully")
```

**Complete Usage Example:**

```gdscript
# Get current stats
var stats = await HStats.get_stats_async()
print("Current stats:")
for stat in stats:
    print("  ", stat.name, ": ", stat.value)

# Update a stat
var result = await HStats.ingest_stat_async("wins", 1)
if EOS.is_success(result):
    print("Win stat incremented")
    
    # Fetch updated stats
    stats = await HStats.get_stats_async()
    for stat in stats:
        if stat.name == "wins":
            print("New wins value: ", stat.value)
```

<Note>
  Stats must be configured in the Epic Developer Portal before they can be used. The stat names used in your code must match the names configured in the portal.
</Note>

<Tip>
  Stats are commonly used with achievements. When you ingest a stat, it can automatically unlock achievements that have stat thresholds configured.
</Tip>
