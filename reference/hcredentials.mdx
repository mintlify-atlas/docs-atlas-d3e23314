## Overview

The `HCredentials` class stores the credentials required by Epic Online Services. All credential values are obtained from the [Epic Games Developer Portal](https://dev.epicgames.com/portal/).

## Properties

### product_name

```gdscript
var product_name: String
```

Game name (max 64 characters).

**Example:** `"My Awesome Game"`

### product_version

```gdscript
var product_version: String
```

Game version.

**Example:** `"1.0"`, `"1.2.3"`

### product_id

```gdscript
var product_id: String
```

Product ID of the game, provided by Epic Games.

### sandbox_id

```gdscript
var sandbox_id: String
```

Sandbox ID of the game, provided by Epic Games.

### deployment_id

```gdscript
var deployment_id: String
```

Deployment ID of the game, provided by Epic Games.

### client_id

```gdscript
var client_id: String
```

Client ID to use for the game, provided by Epic Games.

### client_secret

```gdscript
var client_secret: String
```

Client Secret to use for the game, provided by Epic Games.

<Warning>
  Keep your `client_secret` secure. Do not commit it to version control or expose it in client builds.
</Warning>

### encryption_key

```gdscript
var encryption_key: String
```

(Optional) 256-bit Encryption Key for file encryption in hexadecimal format (64 hex characters).

## Usage Example

### Basic Setup

```gdscript
var creds = HCredentials.new()
creds.product_name = "My Game"
creds.product_version = "1.0.0"
creds.product_id = "abc123def456"
creds.sandbox_id = "xyz789uvw012"
creds.deployment_id = "deploy123"
creds.client_id = "client_abc123"
creds.client_secret = "secret_xyz789"
creds.encryption_key = "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef"

# Use with HPlatform
var success = await HPlatform.setup_eos_async(creds)
if success:
    print("EOS initialized successfully")
```

### Loading from Configuration File

```gdscript
# Load credentials from a JSON file (not included in version control)
func load_credentials() -> HCredentials:
    var file = FileAccess.open("user://eos_credentials.json", FileAccess.READ)
    if not file:
        print("Failed to load credentials")
        return null
    
    var json_string = file.get_as_text()
    file.close()
    
    var json = JSON.new()
    var parse_result = json.parse(json_string)
    if parse_result != OK:
        print("Failed to parse credentials JSON")
        return null
    
    var data = json.data
    
    var creds = HCredentials.new()
    creds.product_name = data.get("product_name", "")
    creds.product_version = data.get("product_version", "")
    creds.product_id = data.get("product_id", "")
    creds.sandbox_id = data.get("sandbox_id", "")
    creds.deployment_id = data.get("deployment_id", "")
    creds.client_id = data.get("client_id", "")
    creds.client_secret = data.get("client_secret", "")
    creds.encryption_key = data.get("encryption_key", "")
    
    return creds
```

### Using Environment Variables (for Development)

```gdscript
func load_credentials_from_env() -> HCredentials:
    var creds = HCredentials.new()
    creds.product_name = OS.get_environment("EOS_PRODUCT_NAME")
    creds.product_version = OS.get_environment("EOS_PRODUCT_VERSION")
    creds.product_id = OS.get_environment("EOS_PRODUCT_ID")
    creds.sandbox_id = OS.get_environment("EOS_SANDBOX_ID")
    creds.deployment_id = OS.get_environment("EOS_DEPLOYMENT_ID")
    creds.client_id = OS.get_environment("EOS_CLIENT_ID")
    creds.client_secret = OS.get_environment("EOS_CLIENT_SECRET")
    creds.encryption_key = OS.get_environment("EOS_ENCRYPTION_KEY")
    
    return creds
```

## Getting Your Credentials

1. Log in to the [Epic Games Developer Portal](https://dev.epicgames.com/portal/)
2. Select your organization and product
3. Navigate to **Product Settings**
4. Find your credentials under:
   - **Product ID**: Product Settings > Product
   - **Sandbox ID**: Product Settings > Sandboxes
   - **Deployment ID**: Product Settings > Deployments
   - **Client ID & Secret**: Product Settings > Clients
   - **Encryption Key**: Product Settings > Encryption

<Tip>
  Create separate deployments and sandboxes for development, staging, and production environments.
</Tip>

<Note>
  You need to configure at least one Client Policy in the Developer Portal before you can use your Client ID and Secret.
</Note>
