## Overview

EOSGMultiplayerPeer implements Godot's `MultiplayerPeerExtension` interface using the EOS P2P Interface. It enables direct peer-to-peer connections between players over the internet with automatic NAT punchthrough and relay server fallback.

**Extends:** `MultiplayerPeerExtension`

**Source Files:**
- `eosg_multiplayer_peer.h:17`
- `eosg_multiplayer_peer.cpp:1`

## Connection Modes

EOSGMultiplayerPeer supports three connection modes:

### Server Mode
- Opens a socket and actively listens for incoming connection requests
- Can accept or reject connection requests (auto-accept by default)
- Always has peer ID `1`
- Supports packet relaying to connected clients

### Client Mode
- Can only connect to a server instance
- Cannot accept connection requests from others
- Can only be connected to a single peer (the server)
- Receives unique peer ID from server upon connection

### Mesh Mode
- Allows arbitrary connections between mesh instances
- Can also connect to a server instance
- All mesh instances must use the same socket ID to connect
- Packets are NOT relayed - can only send to directly connected peers
- Each mesh instance has a unique peer ID

## Usage Examples

### Creating a Server

```gdscript
extends Node

var peer: EOSGMultiplayerPeer

func _ready():
    peer = EOSGMultiplayerPeer.new()
    peer.peer_connected.connect(_on_peer_connected)
    peer.peer_disconnected.connect(_on_peer_disconnected)
    peer.incoming_connection_request.connect(_on_connection_request)
    
    var result = peer.create_server("my_game_server")
    if result == OK:
        multiplayer.multiplayer_peer = peer
        print("Server started")

func _on_peer_connected(id: int):
    print("Peer connected: ", id)
    print("User ID: ", peer.get_peer_user_id(id))

func _on_peer_disconnected(id: int):
    print("Peer disconnected: ", id)

func _on_connection_request(data: Dictionary):
    print("Connection request from: ", data["remote_user_id"])
    # Auto-accept is enabled by default, but you can manually control it:
    # peer.accept_connection_request(data["remote_user_id"])
```

### Creating a Client

```gdscript
extends Node

var peer: EOSGMultiplayerPeer

func connect_to_server(server_user_id: String):
    peer = EOSGMultiplayerPeer.new()
    peer.peer_connected.connect(_on_connected_to_server)
    peer.peer_connection_closed.connect(_on_disconnected)
    
    var result = peer.create_client("my_game_server", server_user_id)
    if result == OK:
        multiplayer.multiplayer_peer = peer
        print("Connecting to server...")

func _on_connected_to_server(id: int):
    if id == 1:  # Server always has ID 1
        print("Connected to server!")
        print("My peer ID: ", peer.get_unique_id())

func _on_disconnected(data: Dictionary):
    print("Disconnected from server")
    print("Reason: ", data["reason"])
```

## Important Notes

- The local user ID must be set before creating any multiplayer instance. This is typically done automatically when logging in via the Connect interface.
- Socket IDs must be alphanumeric only and cannot exceed 32 characters.
- EOS handles NAT punchthrough automatically. If a direct connection fails, a relay server is used.
- Server instances always have peer ID `1`.
- Mesh instances do NOT relay packets - peers can only send to directly connected peers.
- The `TRANSFER_MODE_UNRELIABLE_ORDERED` mode is not supported by EOS and will automatically fall back to `TRANSFER_MODE_RELIABLE`.
- Maximum packet size is limited by `EOS_P2P_MAX_PACKET_SIZE` (approximately 1170 bytes).
- Connection requests are auto-accepted by default. Use `set_auto_accept_connection_requests(false)` for manual control.
