## Overview

HPlatform is responsible for initializing the EOS SDK and creating the EOS platform instance. This is the first component you need to set up before using any other EOS features.

## Methods

### setup_eos_async

```gdscript
func setup_eos_async(p_creds: HCredentials) -> bool
```

Easy setup for EOS. Returns `true` if EOS setup is successful.

**Parameters:**
- `p_creds`: [HCredentials](/api/hplatform#hcredentials) object containing your Epic Games credentials

**Returns:** `true` on success, `false` on failure

**Example:**
```gdscript
var creds = HCredentials.new()
creds.product_name = "My Game"
creds.product_version = "1.0"
creds.product_id = "YOUR_PRODUCT_ID"
creds.sandbox_id = "YOUR_SANDBOX_ID"
creds.deployment_id = "YOUR_DEPLOYMENT_ID"
creds.client_id = "YOUR_CLIENT_ID"
creds.client_secret = "YOUR_CLIENT_SECRET"

var success = await HPlatform.setup_eos_async(creds)
if success:
    print("EOS initialized successfully!")
```

<Note>
This method automatically retries initialization and platform creation if they fail. It combines both `initialize_async()` and `create_platform_async()` calls.
</Note>

## HCredentials

Credentials required by Epic Online Services. Get these values from the [Dev Portal](https://dev.epicgames.com/portal/en-US/).

### Properties

- `product_name: String` - Game name (max 64 characters), e.g., "My Awesome Game"
- `product_version: String` - Game version, e.g., "1.0"
- `product_id: String` - Product ID of the game, provided by Epic Games
- `sandbox_id: String` - Sandbox ID of the game, provided by Epic Games
- `deployment_id: String` - Deployment ID of the game, provided by Epic Games
- `client_id: String` - Client ID to use for the game, provided by Epic Games
- `client_secret: String` - Client Secret to use for the game, provided by Epic Games
- `encryption_key: String` - (Optional) 256-bit Encryption Key for file encryption in hexadecimal format (64 hex characters)

## Best Practices

<Warning>
Always initialize the platform before attempting to use any other EOS features. The platform must be created successfully before you can authenticate users or use other services.
</Warning>

1. **Set appropriate tick budget**: Configure `tick_budget_in_milliseconds` based on your target FPS to ensure EOS has enough time to process events
2. **Handle log messages**: Connect to the `log_msg` signal to capture EOS SDK logs for debugging
3. **Use setup_eos_async**: For most cases, use `setup_eos_async()` instead of manually calling `initialize_async()` and `create_platform_async()`
