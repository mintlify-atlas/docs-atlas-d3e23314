## Overview

HP2P provides methods to configure and query peer-to-peer networking settings including NAT type, relay control, port ranges, and packet queue information.

## Usage Example

```gdscript
extends Node

func _ready():
    await configure_p2p()

func configure_p2p():
    # Check NAT type
    var nat_type = await HP2P.get_nat_type_async()
    print("NAT Type: ", EOS.P2P.NATType.find_key(nat_type))
    
    # Configure relay servers based on NAT type
    match nat_type:
        EOS.P2P.NATType.Moderate, EOS.P2P.NATType.Strict:
            # Use relays for difficult NAT types
            HP2P.set_relay_control(EOS.P2P.RelayControl.AllowRelays)
            print("Relay servers enabled")
        _:
            # No relays for open NAT
            HP2P.set_relay_control(EOS.P2P.RelayControl.NoRelays)
            print("Direct connections only")
    
    # Configure port range
    var result = HP2P.set_port_range(7777, 10)
    if EOS.is_success(result):
        print("Port range configured: 7777-7787")
    
    # Set packet queue sizes (1MB each)
    HP2P.set_packet_queue_size(1024 * 1024, 1024 * 1024)

func monitor_packet_queues():
    var queue_info = HP2P.get_packet_queue_info()
    if EOS.is_success(queue_info.result_code):
        var incoming_usage = float(queue_info.incoming_packet_queue_current_size_bytes) / queue_info.incoming_packet_queue_max_size_bytes
        var outgoing_usage = float(queue_info.outgoing_packet_queue_current_size_bytes) / queue_info.outgoing_packet_queue_max_size_bytes
        
        print("Incoming queue: %.1f%% full" % (incoming_usage * 100))
        print("Outgoing queue: %.1f%% full" % (outgoing_usage * 100))
        
        if incoming_usage > 0.8 or outgoing_usage > 0.8:
            push_warning("Packet queues nearly full!")
```

## NAT Types

The NAT type affects connection quality:

- `Unknown` - NAT type is unknown
- `Open` - Can connect to all NAT types
- `Moderate` - Can connect to Open and Moderate
- `Strict` - Can only connect to Open

## Relay Control Options

- `NoRelays` - Never use relay servers (best performance, may fail with strict NAT)
- `AllowRelays` - Use relay servers as fallback (recommended)
- `ForceRelays` - Always use relay servers (guaranteed connectivity, higher latency)

## Best Practices

1. **Check NAT type**: Query NAT type at startup to determine relay strategy
2. **Allow relays**: Use `AllowRelays` for best balance of performance and connectivity
3. **Configure ports**: Set a reasonable port range to avoid conflicts
4. **Monitor queues**: Periodically check packet queue usage to detect network issues
5. **Adjust queue sizes**: Increase queue sizes for high-traffic games

<Note>
P2P settings should be configured before establishing any peer connections.
</Note>

<Warning>
Using `ForceRelays` will always route traffic through Epic's relay servers, which adds latency. Only use this for testing or when direct connections are not possible.
</Warning>
