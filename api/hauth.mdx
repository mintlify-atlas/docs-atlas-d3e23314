## Overview

HAuth manages user authentication for both Epic Account Services (EAS) and Epic Game Services (EGS). It supports multiple login methods including DevTool, Account Portal, Epic Games Launcher, persistent auth, and anonymous login.

## Methods

### login_devtool_async

```gdscript
func login_devtool_async(server_url: String, credential_name: String) -> bool
```

Login using the Epic Dev Auth tool.

**Parameters:**
- `server_url`: The URL of the dev auth server
- `credential_name`: The credential name to use

**Returns:** `true` on success, `false` on failure

**Example:**
```gdscript
var success = await HAuth.login_devtool_async("localhost:6547", "TestUser")
if success:
    print("Logged in as: ", HAuth.display_name)
```

```gdscript

```

Login using credentials provided by the Epic Games Launcher.

To test this locally, provide the CLI argument `-AUTH_PASSWORD=<exchange_code>` when running your game:
```bash
godot4 . -AUTH_PASSWORD=1234
```

You can generate an exchange code using the DevAuthTool and accessing: `http://localhost:<PORT>/<credential_name>/exchange_code`

**Returns:** `true` on success, `false` on failure

### login_async

```gdscript
func login_async(opts: EOS.Auth.LoginOptions) -> bool
```

Login using EOS Auth by either using an Identity provider or Epic Games Account. This is the recommended way of logging in as you get access to Epic Account Services features: Friends, Presence, Social Overlay, ECom, etc.

**Parameters:**

**Returns:** `true` on success, `false` on failure

### login_persistent_auth_async

```gdscript
func login_persistent_auth_async() -> bool
```

Login automatically if an Epic refresh token is stored by a previous login with Epic Account Portal.

**Returns:** `true` on success, `false` on failure

**Example:**
```gdscript

    # Fall back to another login method

```

## Login Flow

<Warning>
MFA (Multi-Factor Authentication) is not supported by the EOS SDK. Use an account without MFA or use an alternative login method.
</Warning>

For more information about EOS login flows, see [Epic's official documentation](https://eoshelp.epicgames.com/s/article/What-is-the-correct-login-flow-for-a-game-that-supports-crossplay).

## Best Practices

1. **Use persistent auth**: Implement `login_persistent_auth_async()` to automatically log users in on subsequent sessions
2. **Handle login errors**: Connect to the `login_error` signal to display appropriate error messages
3. **Fetch external accounts**: Keep `auto_fetch_external_account` enabled to get user display names
4. **Choose the right login method**: Use `login_async()` for full Epic Account Services features, or `login_game_services_async()` for game services only
