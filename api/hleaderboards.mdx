## Overview

HLeaderboards provides methods to query leaderboard definitions and retrieve leaderboard records with rankings.

## Methods

### get_leaderboard_definitions_async

```gdscript
func get_leaderboard_definitions_async() -> Array
```

Returns an array of dictionaries containing all the leaderboards configured for your game.

**Returns:** Array of dictionaries, where each leaderboard has:
- `leaderboard_id: String` - unique id
- `stat_name: String` - name of stat used to rank the leaderboard
- `aggregation: EOS.Leaderboards.LeaderboardAggregation` - aggregation used to sort the leaderboard
- `start_time: int` - unix timestamp for the start time or `EOS.Leaderboard.LEADERBOARD_TIME_UNDEFINED`
- `end_time: int` - unix timestamp for the end time or `EOS.Leaderboard.LEADERBOARD_TIME_UNDEFINED`

**Example:**
```gdscript
var leaderboards = await HLeaderboards.get_leaderboard_definitions_async()
for leaderboard in leaderboards:
    print(leaderboard.leaderboard_id, " - ", leaderboard.stat_name)
```

### get_leaderboard_records_async

```gdscript
func get_leaderboard_records_async(leaderboard_id: String) -> Array
```

Returns an array of dictionaries containing the top records in the leaderboard.

**Parameters:**
- `leaderboard_id`: The ID of the leaderboard to query

**Returns:** Array of dictionaries, where each record has:
- `leaderboard_id: String` - unique id
- `user_id: String` - product user id of user associated with this record
- `rank: int` - sorted position in the leaderboard
- `score: int` - leaderboard score
- `user_display_name: String` - latest display name for the user or empty string if the user doesn't have one

**Example:**
```gdscript
var records = await HLeaderboards.get_leaderboard_records_async("global_score")
for i in range(min(10, records.size())):
    var record = records[i]
    print("#%d: %s - %d points" % [record.rank, record.user_display_name, record.score])
```

Source: [hleaderboards.gd:74](https://github.com/YOUR_REPO/hleaderboards.gd#L74)

<Note>
Leaderboards must be configured in the Epic Games Developer Portal and linked to a stat before they can be queried.
</Note>

## Usage Example

```gdscript
extends Control

@onready var leaderboard_list = $LeaderboardList

func _ready():
    display_leaderboard("global_score")

func display_leaderboard(leaderboard_id: String):
    # Clear existing entries
    for child in leaderboard_list.get_children():
        child.queue_free()
    
    # Fetch leaderboard records
    var records = await HLeaderboards.get_leaderboard_records_async(leaderboard_id)
    
    if records.is_empty():
        var label = Label.new()
        label.text = "No records found"
        leaderboard_list.add_child(label)
        return
    
    # Display top 10
    for i in range(min(10, records.size())):
        var record = records[i]
        var entry = create_leaderboard_entry(record)
        leaderboard_list.add_child(entry)

func create_leaderboard_entry(record: Dictionary) -> HBoxContainer:
    var container = HBoxContainer.new()
    
    # Rank
    var rank_label = Label.new()
    rank_label.text = "#" + str(record.rank)
    rank_label.custom_minimum_size.x = 50
    container.add_child(rank_label)
    
    # Player name
    var name_label = Label.new()
    name_label.text = record.user_display_name if record.user_display_name else "Anonymous"
    name_label.size_flags_horizontal = Control.SIZE_EXPAND_FILL
    container.add_child(name_label)
    
    # Score
    var score_label = Label.new()
    score_label.text = str(record.score)
    score_label.custom_minimum_size.x = 100
    score_label.horizontal_alignment = HORIZONTAL_ALIGNMENT_RIGHT
    container.add_child(score_label)
    
    return container
```

## Multiple Leaderboards

You can have multiple leaderboards for different stats:

```gdscript
func show_all_leaderboards():
    var leaderboards = await HLeaderboards.get_leaderboard_definitions_async()
    
    for leaderboard in leaderboards:
        print("\nLeaderboard: ", leaderboard.leaderboard_id)
        var records = await HLeaderboards.get_leaderboard_records_async(leaderboard.leaderboard_id)
        
        for record in records:
            print("  #%d: %s - %d" % [record.rank, record.user_display_name, record.score])
```

## Aggregation Types

Leaderboards can use different aggregation methods:

- `EOS.Leaderboards.LeaderboardAggregation.Min` - Lowest value wins
- `EOS.Leaderboards.LeaderboardAggregation.Max` - Highest value wins
- `EOS.Leaderboards.LeaderboardAggregation.Sum` - Sum of all values
- `EOS.Leaderboards.LeaderboardAggregation.Latest` - Most recent value

```gdscript
var leaderboards = await HLeaderboards.get_leaderboard_definitions_async()
for leaderboard in leaderboards:
    match leaderboard.aggregation:
        EOS.Leaderboards.LeaderboardAggregation.Min:
            print(leaderboard.leaderboard_id, " - Best time (lowest)")
        EOS.Leaderboards.LeaderboardAggregation.Max:
            print(leaderboard.leaderboard_id, " - High score (highest)")
```

## Best Practices

1. **Configure in Dev Portal**: Set up leaderboards in the Epic Games Developer Portal before using them
2. **Link to stats**: Leaderboards are automatically populated based on the stat they're linked to
3. **Cache definitions**: Load leaderboard definitions once and cache them
4. **Refresh periodically**: Update leaderboard records periodically to show current rankings
5. **Handle empty results**: Always check if the returned array is empty before displaying
6. **Show limited results**: Display only the top N entries to keep UI manageable

<Warning>
Leaderboard rankings update automatically when players update their stats using [HStats.ingest_stat_async()](/api/hstats#ingest_stat_async). You don't need to manually submit scores to leaderboards.
</Warning>
